# Guestbook E2E User Stories and Test Flows

| ID | Test File Name | As | I want | So that | Acceptance Criteria | Journey / Flow | Status |
|---|---|---|---|---|---|---|---|
| 1 | `guest-read-posts-home.spec.js` | Guest | To read posts on the home page | I can view posts from other users | The post list loads correctly on the home page | Open home → View post list → Assert post text is visible | Complete |
| 2 | `user-can-create-posts.spec.js` | User | To create a new post on the home page | my post is placed on the home feed | The new post displays on the home page | Open home → Type in a new post -> click send → Assert new post is visible in the feed | Incomplete |
| 3 | `guest-cannot-create-posts.spec.js` | Guest | To be restricted from making posts | Unverified users cannot create posts | No option is presented to guests for creating new posts | Open home → Assert New Post form is hidden | Complete |
| 4 | `guest-contact-admin-full-form.spec.js` | Guest | To contact the site administrator | I can ask questions or request assistance | The full contact form is presented. Clicking Submit hides the form and displays a thank-you message | Open contact → Fill in form fields → Click Submit → Assert thank-you message | Complete |
| 5 | `user-contact-admin-simple-form.spec.js` | User | To contact the site administrator | I can ask questions through a simplified form | A simplified contact form is presented. Clicking Submit hides the form and displays a thank-you message | Open contact → Fill in simplified form → Click Submit → Assert thank-you message | Complete |
| 6 | `registered-user-login.spec.js` | Registered User | To log in with my email and password | I can access protected features | Entering a valid email and password logs the user in successfully | Open login → Fill in email and password → Click Login → Assert navigation to home → Assert welcome message | Incomplete |
| 7 | `user-view-profile.spec.js` | User | To view my profile | I can review my personal data | The user's profile fields are populated with the correct data | Log in → Open profile → Assert displayed data | Incomplete |
| 8 | `user-edit-profile.spec.js` | User | To edit my profile | I can correct outdated or incorrect information | The user can click Edit, update contact details, and save the new data successfully | Log in → Open profile → Click Edit → Assert edit form is visible → Update fields → Click Submit → Assert changes are saved | Incomplete |
| 9 | `user-cancel-profile-edits.spec.js` | User | To cancel profile edits | I can cancel changes before saving | Clicking Cancel restores the previous profile view without saving changes | Log in → Open profile → Click Edit → Make changes → Click Cancel → Assert changes were not saved | Incomplete |
| 10 | `user-logout.spec.js` | User | To log out | I can securely end my session | Clicking Logout and confirming logs the user out successfully | Log in → Click Logout → Confirm alert → Assert navigation to home → Assert logout message | Incomplete |
| 11 | `user-change-password.spec.js` | User | To change my password | I can update my password | Submitting the password form updates the password successfully | Log in → Open profile → Click "Change Password" → Enter new password fields → Click Submit → Assert password change confirmation | Incomplete |
